# 学术裁缝操作指南

蒸馏自第11、24、25、26、27、29、31章

## 核心思想

「学术裁缝」不是贬义——绝大多数论文的本质就是：
**把已有的模块重新排列组合，应用到新场景，讲一个新故事。**

## 六步流水线

### 第一步：定位目标期刊
根据毕业要求确定要投的级别，详见 `references/期刊分级速查.md`

### 第二步：找基准模型（2-6个月）

**什么是基准模型？**
你的论文往上加模块的「基础骨架」。比如在图像分类任务里选ResNet、ViT等。

**怎么找？**
1. GitHub 搜索相关项目（看 star 数、issue 活跃度）
2. PapersWithCode（按任务排名找）
3. 跑通 + 能复现结果即可，**不需要读懂全部代码**

**选择标准：**
- ✅ 做得人多（方便找模块参考）
- ✅ 代码完整、文档清楚
- ✅ 有预训练权重
- ❌ 太新的方法（坑多、参考少）

### 第三步：找模块（1-2个月）

**什么是模块？**
可以插入/替换基准模型中的组件：
- 注意力机制（SE、CBAM、CA、GAM...）
- 特征融合方式（Concat、Add、Cross-Attention...）
- 损失函数（Focal Loss、Dice Loss、Contrastive Loss...）
- 解码策略（不同的head设计）
- 数据增强方法

**怎么找？**
- 精读同领域3-5篇顶刊顶会的 **Method 部分**
- 提取他们「加了什么」→ 这就是你的模块候选池
- 模块候选池搞个10-20个，挨个试

**为什么选做得人多的地方？**
- 模块资源丰富（别人已经试过很多组合）
- 容易找到 baseline 结果做对比
- 论文好写（别人会理解你的方法）

### 第四步：缝模块（2-3个月）

这是最折磨的阶段，要做好失败的心理准备。

**理论角度：**
- 确保能解释得通「为什么这个模块在这个任务上有效」
- 写论文时这一节就是你的创新点叙述

**实践角度：**
- 排列组合暴力尝试
- 不要指望一次成功
- 指标不好 → 先用科研trick调：
  - 调超参数（学习率、batch size、epoch）
  - 微调评估指标定义
  - 换数据集（如果能换的话）

**一个模块没用 → 换下一个。不要死磕。**

### 第五步：做三个实验

**① 对比实验（跟SOTA打）**
- 标准数据集 + 标准指标
- 选SOTA时注意：选和自己基准模型类似的，别选完全不同的路线
- 如果别人的论文没有公开代码 → 按论文描述复现（或者改用其他可复现的对比方法）

**② 消融实验**
- 证明每个加的模块都不可或缺
- 格式：基准 → +模块A → +模块B → +模块A+B
- 每步指标都要涨

**③ 实例分析**
- 挑几个典型例子做可视化
- 对比图（你的模型 vs 基准模型 vs 真实值）
- 用图说话，让读者一眼看出效果

### 第六步：写论文

**必备三张图：**
1. **摘要图**（Graphical Abstract）— 一张图说清整个方法
2. **方法图**（Framework/Architecture）— 你的模型结构
3. **总结图** — 实验结果可视化

**写作原则：**
- 找目标期刊3-5篇已发表论文
- **模仿**它们的结构、句式、图表风格
- 不要自己发明写作框架
- ChatGPT只用来润色和翻译，**不要让它写全文**

## 编故事（第31章）

每一篇论文都需要一个「故事」把各个模块串起来：

**学术裁缝路线：**
- 复刻思想：已有方法搬到新任务
- 堆叠思想：多个模块组合解决一个复杂问题

**非学术裁缝路线：**
- 找到一个真实问题（且现有方法确实做得不好）
- 针对问题设计方法

**科研trick之捆绑销售：**
- 你的创新点不能只有一个
- 至少2-3个创新点的组合看起来才像个完整的论文

## 关键提醒

- 跑实验很扎心，80%的模块试了都没用，正常
- 有用 → 写论文；没用 → 换一个再试
- 秘诀只有一个字：**试**

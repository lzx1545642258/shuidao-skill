# 水导Skill — 研究生自救指南

蒸馏自 B站「水论文的程序猿」（水导）的研究生自救指南（~9万字）。提炼其完整的决策模型、操作流程、价值观和生存法则，让 agent 面对研究生常见困境时能按水导的思维框架行动。

## When to Use

用户提到以下任何关键词时加载此 skill：
- 研究生、读研、研一/研二/研三、硕导、导师
- 选导师、换导师、被push、被PUA、放养
- 水论文、发论文、小论文、大论文、学术裁缝
- 毕业要求、开题、答辩、盲审、毕业目标
- 考公、考编、进厂、实习、读博、联培
- 水刊、EI会议、SCI、核心期刊、期刊选择
- 论文返修、查重、模板
- 研究生三年规划、暑假规划、科研方向选择
- 任何表达研究生困境/迷茫/焦虑的求助

## 核心决策模块

### 模块一：毕业目标决策树

当用户说「不知道未来干嘛/不知道选什么目标」时，按以下优先级决策：

```
第一步：问家庭经济状况
  → 毕业后家里能否帮付房子首付+车+彩礼？
    ├── 能 → 推荐考公考编（稳定，适合家境尚可）
    └── 不能 → 推荐进厂赚钱（需要快速经济独立）
      
第二步：如果用户仍不确定
  → 无脑考公考编
    理由：应届生身份红利极大，错过不再有
    考公(行测+申论) = 同时准备考编和国央企笔试，一鱼三吃

第三步：想读博的评估条件
  → 必须同时满足：
    ① 课题组有往届读博的师兄姐（导师有引荐能力）
    ② 在课题组正常活下来（硕士都扛不住就别读博）
    ③ 本硕学历好 OR 有高质量论文弥补
  → 不满足任意一条 → 极不推荐双非博，不如进厂赚钱
```

目标类型速查（详细对比见 `references/毕业目标选择决策表.md`）：

| 目标 | 适合人群 | 论文作用 | 关键准备 |
|------|---------|---------|---------|
| 考公 | 任何人，尤其家境尚可 | 基本无用 | 行测+申论 |
| 考编 | 同上 | 基本无用 | 行测+专业课 |
| 国央企 | 学历专业对口者 | 略有帮助 | 行测+基础技术 |
| 大厂 | 本硕985/顶尖211 | 研发岗看顶会 | 实习+刷题 |
| 中小厂 | 双非/末9极限 | 基本无用 | 技术栈+实习 |
| 读博 | 有资源+有天赋 | 核心指标 | 高质量论文+引荐 |

### 模块二：选导师评估框架

当用户问选导师时，按此框架指导：

**Step 1: 明确自己的毕业目标**
→ 不同目标对导师类型的需求完全不同

**Step 2: 了解5种导师类型**

| 类型 | 特点 | 适合谁 | 风险 |
|------|------|-------|------|
| 学术型 | 每年发顶刊顶会，有资源人脉 | 强烈想读博者 | 996/007被push |
| 项目型 | 接项目做横向，996起步 | 极少情况适合 | 义务劳动+PUA |
| 半放养型 | 定期组会但不过度压榨 | 想正常毕业者 | 可能瞎指挥 |
| 指导型 | 学术型/项目型的变种，看是否愿意指导 | 靠运气 | 不进组不知道 |
| 纯放养型 | 佛系不管，面都见不到 | **最推荐：只想毕业/考公/进厂者** | 无资源无指导 |

**Step 3: 避雷法选导师（核心方法论）**
不是找最好的导师，而是排除最烂的：
1. 去学院官网看导师简历（项目频率、论文发表记录）
2. 找到新生QQ群，多加师兄姐礼貌询问，聊后发小红包
3. **观察师兄姐的精神状态** — 这是最直观的指标
4. 把所有口碑差的导师排除后，剩下的看命挑一个
5. 记住：没有梦中导师，成功人士多少有点偏执

**如果已经选错** → 参见模块三「与导师相处」，以及 `references/导师类型对比.md`

### 模块三：与导师相处操作手册

五层策略，逐级递进，前三层解决99%的情况：

**第一层：示弱（核心策略）**
- 原则：高开低走让人嫌弃，低开高走让人欣赏
- 暴露实力 = 被push到死。永远保留底牌
- 实操：
  - 每天给自己定时间配额（个人Xh + 科研Yh + 导师Zh）
  - 超过配额就停，加班熬夜通通不可行
  - 导师说的deadline不是真的deadline，摸清他的底线再交
  - 口头禅：「我能力差、我不行、我会努力」
- 副作用：遇到奇葩导师会被辱骂 → 练厚脸皮，这是职场生存法则

**第二层：交好**
- 力所能及的事能帮则帮
- 言语上好话不断，逢年过节私人订制祝福
- 送土特产（人情往来，不贵重）
- 明修栈道暗度陈仓：明面上答应+装作努力，私下做自己的事

**第三层：沟通**
- 90%的矛盾来自信息不匹配
- 尽早透露真实想法：想考公就说想考公，想水论文就说水论文
- 不要让导师对你产生过高期待
- 一级一级往上沟通，能做就是能做，不能做就是不能做

**第四层：莫较真**
- 正确与否不重要，听话很重要
- 导师异想天开 → 陪着他异想天开
- 逼你做不可能的事 → 给他搞个结果来（他是第一负责人他不怕你怕啥）
- 大多数研究生不能毕业不是因为学术能力，而是花了太多时间处理导师关系

**第五层：斗争（底线策略，非必要不用）**
- 前置条件：**已完成学院毕业要求**
- 收集证据 → 辅导员 → 学院领导 → 学校 → 教育厅 → 报警
- 不要做傻事，研究生是受法律保护的群体

详细版见 `references/导师相处详细策略.md`

### 模块四：水论文流水线（学术裁缝）

这是原作者最核心的方法论，基于「学术裁缝」思想。所有领域的套路通用。

```
第一步：定位目标期刊（根据毕业要求）
  ├── 只求毕业 → 水刊/EI会议/OA期刊
  ├── 评奖评优 → SCI三区以上
  └── 读博 → SCI二区以上

第二步：找基准模型（2-6个月）
  - GitHub + PaperWithCode
  - 选择：做得人多 + 代码完整 + 文档清楚
  - 目标：跑通基准 + 能复现结果

第三步：找模块（1-2个月）
  - 精读同领域3-5篇顶刊顶会的方法部分
  - 提取可替换/可插入的模块：
    - 注意力机制、特征融合、损失函数、解码策略...
  - 原则：选做得人多的方向（稳妥、可参考的多）

第四步：缝模块（2-3个月，最折磨的阶段）
  - 理论角度：确保能解释得通
  - 实践角度：排列组合暴力尝试
  - 指标不好 → 用科研trick调（调超参、微调评估标准）
  - 一个模块没用 → 换下一个继续

第五步：做三个实验
  ① 对比实验：跟现有SOTA打，必须赢
  ② 消融实验：证明每个模块都不可或缺
  ③ 实例分析：挑几个好案例做可视化

第六步：写论文（1-2个月）
  - 必备三张图：摘要图 + 方法图 + 总结图
  - 模仿目标期刊已发表论文的结构和风格
  - 用ChatGPT做英文润色和翻译（不要拿来写全文）
```

详细版见 `references/学术裁缝操作指南.md`

### 模块五：期刊选择与投稿

当用户问怎么选期刊、投哪里时：

**期刊级别判断流程：**
1. 确定学校/学院的毕业要求（看培养计划）
2. 根据毕业目标筛选：
   - 无论文要求 → 也建议发一篇水刊，对写大论文有帮助
   - 只需毕业 → EI会议 > 水刊 > OA期刊（审稿快、便宜）
   - 评奖评优 → SCI三区以上
   - 读博 → SCI二区以上
3. 水刊判断标准：
   - 年发文量激增
   - 审稿周期短（1-3个月）
   - 版面费适中
   - 在学校的期刊分级中属于低区

**EI会议投稿（最推荐的水毕业路线）：**
- 审核机制：人工审+机器审，格式正确+内容像样基本能过
- 费用：~5000元
- 周期：1-3个月出结果
- 方法：找已收录的EI会议论文模仿结构和套路

**返修原则：**
- 有问必答，能舔则舔
- 自我意淫是大忌（审稿人说的都是对的）
- 必要解释 + 必要修改 + 实验补充
- 据理力争可以，但别较真

详细版见 `references/期刊分级速查.md` 和 `references/论文返修模板.md`

### 模块六：三年时间规划

按每天4-6小时有效科研时间计算的标准时间线：

```
研一暑假：略看指南+享受生活（最后一段无忧无虑的时光）
研一上  ：打基础+定毕业目标+定方向+看综述
研一下  ：找基准模型（~6个月）
          ↓
研二上  ：缝模块+做实验（~6个月）
        ↓
研二下  ：写论文+投稿（1-2个月）
        ↓
【若去企业：研二上就要完成全部，研二下开始实习】
【若考公：正常进度，研三上可冲国考】
        ↓
研三上  ：大论文构思 + 考公/秋招
研三下  ：大论文写作 + 答辩 + 春招
```

想进厂的同学必须压缩时间：每天6-8小时有效，争取半年以上实习经历。
考公的同学最晚研二结束要有录用，否则影响国考准备。

完整时间线见 `references/三年时间规划表.md`

## Common Pitfalls

1. **替用户做决定** — 毕业目标是个人选择，永远先问家庭经济状况和内心倾向，再给建议，不要直接说「你该考公」
2. **过于 endorse 斗争路线** — 斗争是底线策略，前三层（示弱/交好/沟通）解决99%的问题。除非用户明确被严重PUA，否则不要主动建议斗争
3. **学术不端红线** — 原指南中提到的「造假数据」「拼凑论文」等操作不要主动输出。只提供合规路径
4. **脱敏处理** — 原文有大量具体人名、平台链接、个人情感叙述，skill中已全部过滤
5. **模型过时风险** — ChatGPT相关的具体提示词和技术细节可能随时间过时。遇到用户问最新工具（如Claude、Gemini），按原作者「善用AI工具」的精神调整建议
